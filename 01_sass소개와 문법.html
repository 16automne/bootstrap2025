01 sass소개와 문법

SASS - Syntactically Awesome StyleSheets 줄임말

SCSS - Sassy CSS 줄임말

CSS pre-processor(전처리기)로서 css반복을 줄여 시간 절약을 도와주는 css언어의 확장언어

css를 좀 더 효율적으로 작성 & 유지보수하기 쉽게 만들어줌

장) 변수 및 조건문 등의 새로운 문법을 통해 css를 구조화해 작업 가능

단) 코드가 지저분해질 수 있고 코드가 많아질 수록 유지보수 어려움

프로그램 설치: sass공홈 gui, cli방식의 프로그램들 중 편한 프로그램 다운 후 설치

vs code - 별도 확장 프로그램(live css compiler)을 지원 > ectensition 클릭 후 설치 및 사용

1. 문법

SASS 문법
중괄호({})와 세미콜론(;) 필요X
들여쓰기(tab키)를 사용해 코드 구조 표현
구문이 더 간결하고 Python스타일과 비슷
      sass 작성 예시)
      $primary-color:#3498db;
      body
        background:$primary-color:#3498db;
        color:#fff;

      nav
        ul
          margin:0
          padding:0
          list-style:none
        li
          display:inline-block
          margin-right:10px;
SCSS문법
CSS문법과 매우 유사
중괄호({})와 세미콜론(;)을 사용
기존 CSS와 호환성 높음
      scss 작성 예시)
      $primary-color:#3498db;
      body{
        background:$primary-color:#3498db;
        color:#fff;
      }
      nav{
        ul{
          margin:0
          padding:0
          list-style:none
        }
        li{
          display:inline-block
          margin-right:10px;
        }
      }

CSCC v/s SCSS

SCSS권장
CSS와 문법이 거의 같아 기존 css파일과 통합하기 쉬움. 팀원들의 css에 익숙하다면 scss가 더 쉽게 받아들여질 수 있음. 최신 문법과 기능이 주로 scss에서 지원되며 sass는 유지보수용으로만 사용됨. sass 공식 문서에서도 scss 사용을 권장
SASS사용이유
더 간결한 문법을 선호하거나, 기존 sass 스타ㅣㄹ을 유지해야하는 프로젝트에 유리
결론
"SCSS 사용을 권장" > 더 널리 사용되며 유지보수 및 협업에 유리. 최신 기능 지원 및 css와의 호환성도 뛰어남.
